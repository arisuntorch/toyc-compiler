# 编译系统实践报告草稿

## 1. 项目概述

本项目实现了一个 ToyC 到 RISC-V32 汇编的编译器。编译器从标准输入读取
ToyC 源程序，经过词法分析、语法分析、语义环境维护和代码生成，最终向标
准输出写出 RISC-V32 汇编程序。

## 2. 总体架构

编译器采用 C++20 实现，主要模块集中在 `src/main.cpp`：

- Lexer：识别关键字、标识符、整数、运算符、分隔符，并跳过空白和注释。
- Parser：使用递归下降分析 ToyC 文法，构造抽象语法树。
- AST：表示声明、函数、语句和表达式。
- CodeGen：维护符号表、作用域、常量值、栈帧布局，并生成 RISC-V32 汇编。

## 3. 词法分析

词法分析器支持 ToyC 语言中的关键字、标识符、十进制整数、算术运算符、关
系运算符、逻辑运算符、括号、花括号、分号和逗号。空白字符、单行注释和
多行注释会被忽略。

## 4. 语法分析

语法分析器采用递归下降方法。表达式部分按照优先级分层处理：

1. 逻辑或表达式。
2. 逻辑与表达式。
3. 关系和相等性表达式。
4. 加减表达式。
5. 乘除模表达式。
6. 一元表达式。
7. 基本表达式。

这种结构直接对应 ToyC 文法，便于处理运算符优先级和左结合性。

## 5. 语义处理

编译器维护嵌套作用域符号表，用于区分局部变量、全局变量和常量。常量声明
会在编译期求值，后续引用直接替换成立即数。局部变量和函数形参分配在函数
栈帧中，全局变量分配在 `.data` 段。

## 6. 代码生成

代码生成目标为 RISC-V32 汇编。每个函数建立独立栈帧：

- `ra` 和旧 `s0` 保存在栈帧顶部。
- `s0` 作为帧指针。
- 参数和局部变量使用相对 `s0` 的偏移访问。
- 表达式结果统一放入 `a0`。

函数调用按照 RISC-V 调用习惯处理，前八个参数放入 `a0` 到 `a7`，更多参数
放到调用者栈上。

## 7. 控制流

`if-else` 和 `while` 使用自动生成的局部标签和条件跳转实现。`break` 跳转到
当前循环结束标签，`continue` 跳转到当前循环条件判断标签。逻辑与和逻辑或
使用短路求值生成，不会无条件计算右侧表达式。

## 8. 测试

项目包含三个基础测试：

- `tests/basic.tc`：测试全局变量、常量、算术表达式。
- `tests/control.tc`：测试循环、分支、`break`、`continue` 和逻辑表达式。
- `tests/functions.tc`：测试递归函数和超过八个参数的函数调用。

可使用以下命令生成测试汇编：

```sh
make test
```

## 9. 后续优化方向

当前实现优先保证功能正确性，已经实现了常量表达式折叠、立即数加减法和比
较表达式优化、无函数调用表达式的寄存器求值、局部变量到 `s1`-`s11` 的简
单寄存器分配、条件表达式直接分支生成，以及尾递归自调用转循环。后续可以
继续加入：

- 常量传播。
- 死代码删除。
- 公共子表达式消除。
- 更好的寄存器分配。

## 10. 总结

本项目完成了 ToyC 编译器的基本前端和后端，实现了从源程序到 RISC-V32 汇编
的完整编译流程。整体实现以正确性和可维护性为优先目标，为后续性能优化预
留了空间。
